概述

记录用MWeb生成自己的静态网站的过程方便之后查阅

markdown编辑：macOS markdown编辑器MWeb
网页模版：MWeb自带的网页模版site-fondation-book呈现（不支持网站菜单）模版预览

准备工作

在MWeb的文档库新增静态网站分类，命名为my_site
在my_site的上级目录中先rm -rf my_site再git clone [repo] my_site其中[repo]为GitHub Pages的repository（我的上级目录：/Users/yangxijie/Library/Containers/com.coderforart.MWeb3/Data/Documents/themes/Site，我的[repo]：https://github.com/Yang-Xijie/yang-xijie.github.io.git）
然后你应该添加.gitignore，比如不要上传.DS_store
如果你的仓库公开，那么可以再添加如README.md LICENSE等文件。

开始写作

然后就可以开始新建分类（注意只支持一级分类）和markdown文件了！

发布前的准备

只要不换模版，那么选择生成网站就好了，不用清除重新生成。
发布脚本后打开网站好像有bug我这边打开了文件夹，用处也不大就关掉。
脚本如下：

#!/bin/sh

# Please set the name and email. 
# Because MWeb can't get your github global setting.
git config user.name "yxj"
git config user.email "564197835@qq.com"

git add .
# git add --ignore-removal . # 没有使用自带的这句话，因为我希望删除的文件确实被删除掉
git commit -m "{{year}}-{{month}}-{{day}} {{time}}" # 这里的环境变量是可以通过复制传递到终端的
git push -u origin master # 加了参数-u后，以后即可直接用git push 代替git push origin master; -u 的作用其实就在于修改 git push/pull 时默认的 upstream（因为你可能有多个远端仓库）

发布

每次要发布的时候，在MWeb右键选择复制发布脚本并打开终端…，终端打开后复制即可


这时MWeb会调出你在偏好设置中写好的脚本，然后换掉其中的{{time}}等字符串；之后复制这段脚本到剪贴板，然后打开终端。你只需在终端⌘ V粘贴刚刚的那段脚本就好了。
